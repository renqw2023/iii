# 2026-03-11 侧边栏头像弹窗与图像区布局调整开发文档

## 1. 背景

本次需求来自桌面端左侧侧边栏的用户头像弹窗优化，目标有两部分：

1. 在头像点击后的弹窗中加入与首页一致的界面语言选择能力。
2. 解决弹窗底部和右侧被截断的问题，并同步优化图像内容区与侧边栏的整体空间分配。

用户在第一次验收后继续反馈，问题并不只是弹窗本体宽度，而是整体布局关系不合理：

- 头像弹窗加入语言选项后更容易向右撑出。
- 图像区主内容容器四周空白偏大，导致可用宽度被浪费。
- 左侧侧边栏本身偏窄，不利于承载新增语言入口和账户菜单。

因此这次修改最终演变为一次「弹窗 + 侧边栏 + 图像区布局」联合调整。

## 2. 问题复现

### 2.1 初始现象

在桌面端页面中，点击侧边栏底部头像后出现账户弹窗。加入语言选项后，出现以下问题：

- 弹窗右侧被裁掉，语言选择框无法完整显示。
- 弹窗整体与侧边栏底部按钮区域重叠感较强。
- 图像区页面右侧内容容器留白较宽，侧边栏与内容区的空间比例失衡。

### 2.2 用户反馈后的关键判断

用户提供截图和描述后，可以判断问题不只是「弹窗太大」，而是以下三层叠加：

1. 弹窗新增语言控件后，横向占用显著增加。
2. `Sidebar` 组件外层虽然放开了自身溢出，但 `Layout` 中包裹桌面侧边栏的容器仍然在裁切。
3. 图像区页面外层 padding 偏大，让侧边栏与主内容的相对宽度更紧张。

## 3. 根因分析

### 3.1 直接根因：桌面侧边栏外层容器仍然裁切子元素

文件：

- `client/src/components/Layout/Layout.js`

原本桌面侧边栏外层容器使用：

```jsx
className="hidden md:block sticky top-0 h-screen flex-shrink-0 overflow-hidden"
```

即使 `Sidebar` 内部已经允许溢出显示，只要最外层还是 `overflow-hidden`，头像弹窗仍会在该层被裁切。

### 3.2 诱发因素：头像菜单宽度被语言控件横向撑大

文件：

- `client/src/components/Layout/Sidebar.js`

语言入口最初以横向标题 + 右侧下拉框的形式布局，在有限宽度下容易把弹窗撑宽，导致更容易超出侧边栏区域。

### 3.3 空间分配问题：图像区容器 padding 偏大

文件：

- `client/src/styles/gallery.css`

`gallery-page` 与 `seedance-page` 的四边 padding、`gallery-stage` 的内边距，以及 `gallery-main` 的内边距都偏大。这样会让：

- 主内容区域看起来“缩在中间”
- 左侧侧边栏视觉上显得更窄
- 用户更容易感知到弹窗“挤”

## 4. 实现目标

本次最终实现目标拆分为四项：

1. 头像弹窗中加入语言切换能力。
2. 让弹窗完整显示，不再被外层容器裁切。
3. 左侧桌面侧边栏适度加宽，给账户菜单更多安全空间。
4. 图像区页面缩小四边留白，把更多宽度还给内容本身。

## 5. 具体改动

### 5.1 `client/src/components/Layout/Sidebar.js`

#### 5.1.1 接入 i18n 能力

新增：

```jsx
import { useTranslation } from 'react-i18next';
import { Languages } from 'lucide-react';
```

并在组件中注入：

```jsx
const { i18n, t } = useTranslation();
```

同时增加三种运行时支持语言：

```jsx
const languageOptions = [
  { value: 'zh-CN', label: t('language.zh-CN', '简体中文') },
  { value: 'en-US', label: t('language.en-US', 'English') },
  { value: 'ja-JP', label: t('language.ja-JP', '日本語') },
];
```

#### 5.1.2 在头像弹窗中加入语言选择项

在账户菜单的常规导航项和退出按钮之间加入一个新的语言区块：

- 标题行使用 `Languages` 图标 + `Select language`
- 下方是全宽 `select`
- 通过 `i18n.changeLanguage()` 切换语言

最终采用纵向布局，而不是左右布局，原因是纵向布局在小宽度弹窗中更稳定，不会轻易挤出侧边栏边界。

#### 5.1.3 调整侧边栏与弹窗尺寸

侧边栏展开宽度：

- 从 `240` 调整为 `264`

头像弹窗宽度：

- 初始从 `245` 扩大后又回调
- 最终稳定为 `236`

其它视觉细节：

- `borderRadius` 从 `14` 调整为 `16`
- `padding` 从 `6` 调整为 `8`
- 分隔线负边距从 `-6px` 调整到 `-8px`

这些微调的目的，是让弹窗在更窄宽度下依然保持信息密度和视觉平衡。

#### 5.1.4 放开 `Sidebar` 自身溢出

```jsx
className="flex flex-col h-full overflow-visible"
```

这样 `Sidebar` 自己不会再对头像弹窗做裁切。

### 5.2 `client/src/components/Layout/Layout.js`

这是本次真正解决裁切问题的关键修改。

桌面侧边栏外层容器：

- `overflow-hidden` 改为 `overflow-visible`
- 展开宽度从 `240` 改为 `264`

修改后：

```jsx
className="hidden md:block sticky top-0 h-screen flex-shrink-0 overflow-visible"
style={{ width: collapsed ? 64 : 264 }}
```

这一层修改确保头像弹窗即使超出 `Sidebar` 盒模型，也不会在 `Layout` 层被截掉。

### 5.3 `client/src/styles/gallery.css`

为了让主内容区不要保留过宽空白，同时给左侧区域更多视觉空间，对图像区页面做了三组收缩：

#### 5.3.1 页面外层 padding

```css
.gallery-page,
.seedance-page {
  padding: 0.75rem 0.875rem 3rem;
}
```

原来是：

```css
padding: 1.25rem 1.25rem 4rem;
```

效果：

- 上下左右整体更紧凑
- 页面边缘留白明显收窄
- 内容区横向可用宽度增加

#### 5.3.2 Stage 内边距

```css
.gallery-stage {
  padding: 0.875rem;
}
```

原来是 `1.25rem`。

#### 5.3.3 主内容区内边距

```css
.gallery-main {
  padding: 0.875rem;
}
```

原来是 `1.25rem`。

效果是图像内容和筛选布局的总宽度都更高效，不再浪费横向空间。

## 6. 最终结果

完成后，桌面端效果达到以下状态：

1. 头像弹窗中新增了语言切换入口。
2. 语言切换控件不再横向挤压菜单项。
3. 弹窗不再被桌面侧边栏外层容器裁切。
4. 左侧侧边栏展开后更宽，和右侧图像区比例更平衡。
5. 图像区页面四边空白更小，视觉密度更合理。

## 8. 风险与后续建议

### 8.1 当前剩余风险

虽然裁切问题已经从结构上修复，但仍有两个潜在优化点：

1. 头像弹窗当前仍使用绝对定位，如果未来菜单项继续增加，仍可能需要重新分层。
2. `Sidebar.js` 中存在未使用变量，如 `credits`、`isFilterPage`，建议后续清理，避免 warning 持续堆积。

### 8.2 后续建议

建议下一步考虑以下优化：

1. 将头像弹窗抽离为独立的可复用 `AccountMenu` 组件，减少 `Sidebar` 体积。
2. 如果后续菜单项继续增加，可考虑把语言入口做成二级弹层或单独设置入口。
3. 统一桌面侧边栏的宽度常量，避免 `Layout.js` 与 `Sidebar.js` 分别手写数值。

## 10. 总结

本次并不是单纯的“把弹窗调小”，而是一次布局层级修正：

- 从交互层加入语言切换
- 从组件层重排弹窗结构
- 从布局层解除外层裁切
- 从页面层收缩内容区空白

最终解决的是一个由组件尺寸、父级 overflow 和整体页面宽度分配共同造成的问题，而不是单一控件的样式问题。
