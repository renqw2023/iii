# 20260222 项目重新开发需求分析与补充建议

基于我对您提供的 `20260222项目重新开发需求初稿.md` 的理解，并结合当前项目目录的实际技术栈情况（**前端 React + TailwindCSS, 后端 Express.js + Mongoose/MongoDB**），我为您起草了以下需求补充分析报告。

## 1. 现有项目架构匹配
您当前项目的主体框架非常适合本次重构：
* **数据本地化要求**：您拥有 Node.js 后端及 MongoDB 数据支持能力。我们可以通过后端脚本来解析、清洗来自 Github (nano-banana-pro-prompts 和 awesome-seedance-2-prompts) 的数据，实现您的“数据本地化”期望。
* **媒体处理支持**：后端使用了 `fluent-ffmpeg` 依赖，说明已有一定的多媒体基础，这在随后实现 Seedance 2.0 视频功能时可以为我们提供强有力的底层支持。

## 2. 计划中需要补充的技术细节

您的初版需求逻辑非常清晰，为了确保落地效果，我建议在计划中补充以下几点：

### 2.1 数据流与数据库建模补充
初稿中提到了引入多源数据（meigen.ai，nano-banana-pro-prompts 等），但由于这些源的数据结构不同，我们在计划中需要**补充统一的数据结构设计（Schema Design）阶段**。
* 需要针对 Prompts 提取通用字段（如：原图/视频预览、提示词内容、负面提示词、模型版本、分辨率配置等）。
* 明确**“数据本地化”方案**：写一个通用的 Node.js 导入脚本拉取开源项目 repo 的内容。

### 2.2 前端路由与导航架构调整
为保留原有的 Midjourney 风格功能，且新增 2 个完全不同的大板块（Meigen风格提示词区，Seedance视频区），我们需要**补充全局布局设计**。
* 建议对现有的主导航栏（Navbar）进行改版，设定三大主干流（Midjourney / Prompts库 / Seedance视频库）。
* 对于 UI 风格的统一（2.3中提到），需要抽取当前项目的 Tailwind 设计 Token（如主色调主配置、字体、边距），覆写到我们新撸的 meigen.ai 和 seedance 前端组件中。

### 2.3 关于 Seedance 2.0 视频展示方案的优化考量
由于 Seedance 2.0 会涉及到视频（Video）的展示：
* 视频的加载与资源消耗远大于图片，因此计划中建议**补充加入前端懒加载（Lazy Load）、占位缩略图提取或 Intersection Observer 控制视频自动播放/暂停的策略**。
* 存储与带宽：如果是本地化数据，确保目前服务器有足够空间支持（也可以先以“保存视频原URL本地化”而非“原文件本地化”作为初步方案）。

## 3. 下一步工作建议

我已建立好基础的实施大纲与项目代办打卡任务。接下来的流程，我推荐按以下顺序列步进行：

1. **竞品详细剖析**：我开始使用 Browser Agent 深钻 https://www.meigen.ai 和 https://youmind.com/seedance-2-0-prompts ，生成具体的全站功能 UI 复刻切图与分析文档。
2. **制定并确认 API/表结构**：提交新表结构方案供您确认，一旦敲定则开始写导数据脚本。
3. **分版块切入开发**：重构并上线对应页面。
