## 三、踩坑与修复

### 问题：原导入脚本使用顺序编号匹配

**根因**：`importSeedance.js` 解析 README 时用 `currentNumber++` 顺序编号（1, 2, 3...），然后尝试与 `video-urls.json` 的 key（YouMind ID 如 "733", "731"）匹配。两套编号完全对不上，导致所有 529 条记录都回退到占位内容。

**表现**：前端显示 "Seedance Prompt #730"、"Video generation prompt #730" 等占位文本。

**修复**：重写 `parseReadme()` 函数，从每个 `###` section 中提取 `youmind.com/...?id=XXX` 链接中的 ID，用该 ID 作为匹配 key。

```javascript
// 修复前：顺序编号
currentNumber++;
matchedPrompt = parsedPrompts.find(p => p.number === parseInt(promptId));

// 修复后：YouMind ID 匹配
const idMatch = section.match(/youmind\.com[^\)]*[?&]id=(\d+)/);
const youmindId = idMatch[1];
promptMap[youmindId] = { title, prompt, ... };
```

### 其他改进

1. **去重提示词文本**：README 中部分提示词代码块存在重复，使用 `Set` 去重
2. **精选标记**：有完整提示词的 100 条标记为 `isFeatured: true`
3. **增强分类**：扩展 `guessCategory()` 关键词列表，覆盖更多类型
4. **增强标签**：扩展 `extractTags()` 关键词，添加 fpv/drone/pov/cyberpunk 等
5. **`--clear` 参数**：支持先清空数据库再导入
6. **提取 sourceUrl**：带完整提示词的记录使用原文 Source 链接，其他使用 YouMind 链接

# 阶段7.5 — Seedance 视频代理 & 自动同步

**日期**：2026-03-03  
**分支**：main  
**核心变更**：解决 Twitter 视频 CORS 播放问题，添加后端代理 + 前端适配 + YouMind 自动同步脚本

---

## 一、问题背景

| 视频源 | 能播放？ | 有声音？ |
|--------|----------|----------|
| GitHub Releases | ✅ 可跨域播放 | ❌ 无音轨 |
| Twitter/X (twimg.com) | ❌ CORS 限制 | ✅ 有声音 |

**目标**：让 Twitter 视频通过后端代理播放，既能播又有声音。

---

## 二、实现方案

### 1. 后端视频代理接口

`GET /api/seedance/proxy-video?url=xxx`

- 白名单验证域名（`video.twimg.com`、`pbs.twimg.com`、`github.com`）
- 服务器端 `fetch` Twitter 视频并 Stream 转发
- 支持 Range 请求（进度条 seek）
- 设置 `Cross-Origin-Resource-Policy: cross-origin` 允许跨域嵌入
- 缓存 1 小时 (`Cache-Control: public, max-age=3600`)

### 2. 前端代理适配

新增 `getVideoSrc(url)` 工具函数：
```javascript
// client/src/services/seedanceApi.js
export const getVideoSrc = (url) => {
    if (!url) return '';
    if (url.includes('twimg.com')) {
        return `${baseURL}/seedance/proxy-video?url=${encodeURIComponent(url)}`;
    }
    return url;
};
```

`VideoCard.js` 和 `SeedanceDetail.js` 的 `<video src>` 均使用 `getVideoSrc()` 包装。

### 3. YouMind 自动同步脚本

`server/scripts/syncSeedanceFromYouMind.js`

功能：
- 从 YouMind 下载最新 CSV 导出（含完整提示词 + Twitter 视频 URL）
- 智能 Upsert：按 `sourceId` 去重，只更新有变化的字段，保留用户交互数据
- 支持 `--dry-run`（预览不写入）和 `--download-only`（仅下载 CSV）
- CSV 备份到 `_data_sources/seedance/csv_backups/`

```bash
# 预览
node scripts/syncSeedanceFromYouMind.js --dry-run

# 正式同步
node scripts/syncSeedanceFromYouMind.js

# 仅下载 CSV
node scripts/syncSeedanceFromYouMind.js --download-only
```

---

## 三、踩坑与修复

### Bug 1：Express 路由顺序

**问题**：`/proxy-video` 定义在 `/:id` 之后，Express 把 `proxy-video` 当作 `:id` 参数匹配，导致代理接口永远不会被触发。

**修复**：将 `/proxy-video` 路由移到 `/:id` 之前。

```javascript
// ❌ 错误顺序（原来）
router.get('/:id', ...)     // 匹配 "proxy-video" 作为 id
router.get('/proxy-video', ...) // 永远不会到达

// ✅ 正确顺序（修复后）
router.get('/proxy-video', ...) // 精确匹配优先
router.get('/:id', ...)          // 通配在后
```

### Bug 2：跨域资源策略

**问题**：前端 (port 3100) 请求后端 (port 5500) 的代理视频时，浏览器报 `ERR_BLOCKED_BY_RESPONSE.NotSameOrigin`。

**修复**：在代理响应中添加：
```javascript
res.setHeader('Cross-Origin-Resource-Policy', 'cross-origin');
res.setHeader('Access-Control-Allow-Origin', '*');
```

# 阶段8 — Seedance 瀑布流布局 & 移动端适配

**日期**：2026-03-03  
**分支**：main  
**核心变更**：将 `/seedance` 页面从传统分页改为无限滚动瀑布流，完善全端响应式布局

---

## 一、背景与目标

### 原有状态

`/seedance` 页面使用 `useQuery` + 传统分页（12条/页，底部 Previous/Next 按钮），网格为固定 3 列 (`repeat(3, 1fr)`)，移动端仅有 ≤900px (2列) 和 ≤600px (1列) 两个断点。

而 `/gallery` 页面已实现 `useInfiniteQuery` + IntersectionObserver 无限滚动瀑布流体验。

### 新需求

让 `/seedance` 页面获得与 `/gallery` 一致的瀑布流无限滚动体验，并完善移动端适配。

---

## 二、实现方案

### 1. SeedanceList.js — 无限滚动重构

| 改动项 | 修改前 | 修改后 |
|--------|--------|--------|
| 数据查询 | `useQuery` | `useInfiniteQuery` |
| 分页组件 | Previous/Next 按钮 | 移除，改为 IntersectionObserver sentinel |
| 每页数量 | 12 条 | 24 条 |
| 数据合并 | 单页 `prompts` | `flatMap` 合并所有已加载页 |
| 加载提示 | 无 | 底部加载动画 + 到底标记 |

核心代码（参考 `GalleryList.js`）：

```javascript
// IntersectionObserver — 滚动到底部时自动加载下一页
useEffect(() => {
    const el = sentinelRef.current;
    if (!el) return;
    const observer = new IntersectionObserver(
        (entries) => {
            if (entries[0].isIntersecting && hasNextPage && !isFetchingNextPage) {
                fetchNextPage();
            }
        },
        { rootMargin: '300px' }
    );
    observer.observe(el);
    return () => observer.disconnect();
}, [hasNextPage, isFetchingNextPage, fetchNextPage]);
```

### 2. gallery.css — 瀑布流网格 + 响应式断点

`.seedance-grid` 从 3 列固定布局升级为与 `.gallery-grid` 一致的全端适配：

| 断点 | 列数 | 间距 |
|------|------|------|
| ≥2400px | 6 列 | 12px |
| ≥1800px | 5 列 | 12px |
| 默认 (桌面) | 4 列 | 12px |
| ≤1200px | 3 列 | 12px |
| ≤900px | 2 列 | 8px |
| ≤600px | 2 列 | 6px |
| ≤400px | 1 列 | 0 |

```css
/* 修改前 */
.seedance-grid {
  grid-template-columns: repeat(3, 1fr);
  gap: 1.25rem;
}

/* 修改后 */
.seedance-grid {
  grid-template-columns: repeat(4, 1fr);
  gap: 12px;
}
```

### 3. VideoCard.js — 懒加载优化

将 IntersectionObserver 的 `threshold` 从 `0.3` 降低到 `0.1`，让视频在瀑布流中更早开始加载（因为卡片更多、可视面积更小）。

# 阶段9：视频缩略图优化 & 详情页来源/翻译 (2026-03-03)

## 一、问题背景

用户反馈当前 `/seedance` 页面存在以下问题：

1. **首屏黑框**：529+ 个视频使用 `preload="metadata"`，浏览器需为每个视频建立网络连接获取元数据才能显示第一帧，导致用户打开页面首先看到大量黑色方框
2. **详情页缺少来源**：数据库中已有 `sourceUrl` 和作者信息，但详情页 UI 未展示
3. **详情页缺少翻译**：用户需要将英文提示词翻译成中文，参考 YouMind 的"翻译前/翻译后"切换功能

## 二、解决方案

### 2.1 缩略图优化 — 三层渲染架构

**核心思路**：用「骨架屏 → 缩略图/首帧 → 悬停播放视频」三层叠加替代纯黑框等待。

#### 三层渲染：

| 层级 | Z-Index | 元素 | 触发条件 |
|------|---------|------|----------|
| 底层 | 1 | 渐变骨架屏动画 | 始终存在，缩略图加载完毕后隐藏 |
| 中层 | 2 | 缩略图/首帧 | 有 `thumbnailUrl` 用 `<img>`；无则用 `<video preload="metadata">` 提取首帧 |
| 顶层 | 3 | 播放视频 | 仅 hover 时加载视频 `src` 并播放 |

#### 关键优化：
- `preload` 从 `"metadata"` 改为 `"none"`（主视频层），避免 500+ 并发元数据请求
- 仅在 `hover` 时才设置视频 `src`，按需加载
- 有 `thumbnailUrl` 的记录通过后端代理 `/api/seedance/proxy-thumbnail` 加载 Twitter 缩略图（24h 缓存）
- 无 `thumbnailUrl` 的记录使用隐藏的 `<video preload="metadata">` 逐步提取首帧

### 2.2 详情页来源 & 作者信息

在详情页添加两个高亮按钮：
- **作者按钮**（金色渐变）：显示 `authorName`，点击跳转 `authorLink`
- **来源按钮**（绿色渐变）：显示"来源"文字，点击跳转 `sourceUrl`

### 2.3 翻译切换功能

- 详情页提示词区域 header 添加「翻译/翻译前」切换按钮
- 后端新增 `POST /api/seedance/:id/translate` 路由
- 使用 Google Translate 免费 API (`translate.googleapis.com`) 进行翻译
- 翻译结果缓存到数据库 `translations` Map 字段，避免重复调用
- 翻译按钮带 loading spinner 动画
