# 首页 T 字形布局修复开发日志

**日期**: 2026-03-06
**分支**: main
**涉及文件**: `client/src/styles/gallery.css`

---

## 问题描述

首页呈现明显的"T 字形"布局：
- Hero 横幅区域铺满全屏宽度
- Hero 下方的"Explore Our Collections"和"Latest Content"两个区块被 `max-width: 1400px` + `margin: 0 auto` 强制居中
- 在宽屏（1920px+）下，两侧各留出约 **260px** 的空白，视觉上与全宽 Hero 严重割裂

## 根本原因分析

```css
/* 修改前 */
.home-section {
  max-width: 1400px;   /* 硬性宽度上限 */
  margin: 0 auto 1.5rem;  /* 强制居中，产生大量边距 */
  padding: 0 1.5rem;
}

.home-content-section {
  max-width: 1400px;
  margin: 0 auto 2rem;
  /* ... */
}
```

`home-dark-area` 容器本身只有 `1.25rem` 的左右 padding，但内层 section 的 `max-width: 1400px` 在宽屏上造成巨大的自动边距。

## 修复方案

**策略**：移除内层 section 的 `max-width` 限制，改为让 section 填满容器宽度；由外层 `home-dark-area` 统一控制左右间距（从 `1.25rem` 调整为 `2.5rem`）。

```css
/* 修改后 */
.home-dark-area {
  padding: 2rem 2.5rem 4rem;   /* 从 1.25rem 增至 2.5rem，统一左右呼吸感 */
}

.home-section {
  width: 100%;         /* 填满容器，无 max-width 限制 */
  margin: 0 0 1.5rem;
  padding: 0;          /* 间距由 home-dark-area 控制 */
}

.home-content-section {
  width: 100%;
  margin: 0 0 2rem;
  /* 去掉 max-width: 1400px */
}
```

响应式断点也同步调整：
- `@media (max-width: 768px)`: `home-dark-area` padding 改为 `1rem`，section padding 归零
- `@media (max-width: 480px)`: `home-dark-area` padding 改为 `0.75rem`，section padding 归零

## 效果对比

| 项目 | 修复前 | 修复后 |
|------|--------|--------|
| 1920px 屏每侧空白 | ~260px | ~40px（2.5rem） |
| 布局形状 | T 字形 | 矩形均衡 |
| Latest Content 宽度 | ~1400px 居中 | 近全宽 |
| Hero 与内容对齐感 | 割裂 | 一致 |

## 验证

- 浏览器截图确认：Explore Collections 和 Latest Content 两个区块现在均接近全宽，与 Hero 对齐
- Chrome DevTools 控制台：无任何 JS 错误或 CSS 警告
- 测试地址：`http://localhost:3100`

## 影响评估

- 仅修改 CSS，不涉及 JS 逻辑或组件结构
- 响应式布局（768px、480px）同步更新，移动端不受影响
- 其他页面（gallery、seedance 等）使用独立 class，不受此次修改波及
