## 一、问题描述

### 问题 1：图片下部白色容器溢出 + 平角

**现象**：首页 Latest Content 区域和 /gallery 页面的卡片图片底部出现一条白色横条，且图片底角呈平角（与对标站 meigen.ai 的全圆角展示风格不符）。

**复现路径**：浏览器默认亮色主题，打开首页或 /gallery 页面，滚动到图片卡片区域即可看到每张图片底部有约 8px 的白色背景条，图片四角本该为圆角但底部两角显示为平角。

### 问题 2：首页 Latest Content 关闭弹窗后滚回顶部

**现象**：用户在首页 Latest Content 区域浏览图片，点击图片打开详情 Modal（跳转至 `/explore/:id`），查看完毕后点击右上角 X 按钮关闭，页面返回首页但滚动位置跳回顶部，而非用户离开时所在的位置。

**复现路径**：在首页滚动至 1500px 以下 → 点击任意卡片 → 在 SrefModal 中点击 X 关闭 → 首页恢复后 scrollY ≈ 0（顶部）。

---

## 二、根因分析

### 根因 1：Masonry 行间隙导致卡片容器底部"白色溢出"

**技术背景**：
首页 SrefCard 和 /gallery GalleryCard 均采用 CSS Grid + `grid-auto-rows: 8px` 的 Masonry 布局方案。每张卡片通过 `gridRowEnd: span N` 控制高度，其中 `N` 的计算公式为：

```js
const ROW_HEIGHT = 8;
const ROW_GAP = 8;
setGridSpan(Math.ceil((renderedH + ROW_GAP) / ROW_HEIGHT));
```

`ROW_GAP` 额外加 8px 是为了在 row-gap=0 的 grid 中模拟卡片之间的视觉间距。这导致 **卡片高度 = 图片高度 + 8px**。

**白色溢出的产生**：
- `.liblib-card` / `.gallery-card` 的 `background` 在亮色主题下为 `var(--gallery-card-bg) = #ffffff`
- 图片只占卡片高度的前 N-1 行（即卡片高度 - 8px），多出来的 8px 显示的是卡片的 `#ffffff` 背景色
- 结果：每张图片底部有一条 8px 的白色横条

**平角的产生**：
- `.liblib-card` / `.gallery-card` 有 `border-radius: 12px` + `overflow: hidden`，理论上会把内部图片四角裁成圆角
- 但卡片的 `border-radius` 仅在卡片边缘最后 12px 处产生圆弧裁切
- 图片在卡片底部 12px 区域内只占 4px（最后 8px 是白色底条），实际横向裁切量约 0.7px，肉眼观察图片底角为平角
- 而白色底条的底部（卡片真正底部）有完整的 12px 圆角，视觉上是"圆角的白条在图片下方"

**简化示意**：
```
[图片区域 - 200px，底角被裁切 < 1px，近似平角]
[白色底条 - 8px，底角有完整 12px 圆角]
= 图片平角 + 白色容器溢出
```

**对标站 meigen.ai 的做法**：直接对图片容器设置 `border-radius`，不依赖父容器的 `overflow: hidden` 来做圆角裁切，且父容器无背景色。

### 根因 2：Home.js unmount 时 scrollY 已被浏览器压缩

**问题链条**：

```
用户在首页 scrollY = 1500
     ↓ 点击 SrefCard → navigate('/explore/:id', { state: { fromList:true } })
     ↓
React Router 开始路由切换：卸载 Home，挂载 Explore + SrefModal
     ↓
浏览器 DOM 替换为 Explore 页面内容（isListActive=false，页面极短）
     ↓
浏览器将 window.scrollY 压缩至新页面最大可滚动值 ≈ 42px
     ↓
Home.js unmount cleanup 运行：
  sessionStorage.setItem('home_scrollY', String(Math.round(window.scrollY)))
  → 存入 42，而非 1500
     ↓
用户关闭 Modal → navigate(-1) → Home 重新挂载
     ↓
恢复 scrollY = 42（近顶部）而非 1500
```

**关键误判**：原始代码假设"组件 unmount 时 `window.scrollY` 仍是用户离开时的值"，但 React Router 在 commit 阶段替换 DOM 后（新页面高度极短），浏览器即时限制 scrollY，而 cleanup 函数在此之后才执行。

**验证数据**：
```json
// 修复前
{ "scrollY": 1500, "savedBefore": "42" }   // 用户在 1500px，但 cleanup 存了 42

// 修复后
{ "scrollY": 1500, "savedBefore": "1500" } // scroll 事件实时存储，值正确
```

## 三、修复方案

### 修复 1：卡片容器透明化 + 圆角下移至图片容器

**策略**：将视觉样式从"定位容器"（`.liblib-card` / `.gallery-card`）剥离，移至"图片容器"（`.liblib-card-image` / `.gallery-card-image`）。定位容器负责 grid span 定位，图片容器负责视觉呈现。

#### `client/src/components/Post/LiblibStyleCard.css`

```css
/* 修改前 */
.liblib-card {
  background: var(--gallery-card-bg, rgba(255,255,255,0.04));
  border: 1px solid var(--gallery-card-border, rgba(255,255,255,0.08));
  border-radius: 12px;
  overflow: hidden;
  transition: box-shadow 0.25s, border-color 0.25s;
  display: block;
  cursor: pointer;
}
.liblib-card:hover {
  box-shadow: 0 4px 20px rgba(99, 102, 241, 0.15);
}
.liblib-card-image {
  position: relative;
  overflow: hidden;
  background: var(--bg-secondary);
}

/* 修改后 */
.liblib-card {
  background: transparent;           /* 不再有白色背景 */
  border: none;                      /* 去掉容器边框 */
  border-radius: 12px;
  overflow: hidden;
  transition: box-shadow 0.25s;
  display: block;
  cursor: pointer;
}
.liblib-card-image {
  position: relative;
  overflow: hidden;
  border-radius: 12px;               /* 新增：图片容器直接圆角 */
  background: var(--bg-secondary);
  transition: box-shadow 0.25s;      /* 新增：配合 hover shadow */
}
.liblib-card:hover .liblib-card-image {
  box-shadow: 0 4px 20px rgba(99, 102, 241, 0.15);  /* hover 效果移至图片容器 */
}
```

#### `client/src/styles/gallery.css`

**效果**：
- 8px 行间隙变为透明（显示父容器背景），白色溢出消失
- 图片容器自身有 `border-radius: 12px` + `overflow: hidden`，四角均完整圆角
- hover 阴影作用于实际可见的图片区域，视觉效果更精准

### 修复 2：改为 scroll 事件实时持久化滚动位置

**策略**：不依赖 unmount cleanup 保存 scrollY，改为 scroll 事件监听器在用户滚动时实时写入 sessionStorage。用户点击卡片导航前，sessionStorage 里已有正确值，完全不受 unmount 时机影响。

#### `client/src/pages/Home.js`

```js
// 修改前：unmount 时保存（timing 不可靠）
useEffect(() => {
  return () => {
    sessionStorage.setItem(HOME_SCROLL_KEY, String(Math.round(window.scrollY)));
  };
}, []);

// 修改后：scroll 事件实时保存（timing 可靠）
useEffect(() => {
  const save = () => sessionStorage.setItem(HOME_SCROLL_KEY, String(Math.round(window.scrollY)));
  window.addEventListener('scroll', save, { passive: true });
  return () => window.removeEventListener('scroll', save);
}, []);
```

**scroll 事件恢复逻辑不变**（阶段13已实现）：
```js
useEffect(() => {
  if (navigationType !== 'POP') return;
  if (isLoading) return;
  const saved = sessionStorage.getItem(HOME_SCROLL_KEY);
  if (!saved) return;
  sessionStorage.removeItem(HOME_SCROLL_KEY);
  requestAnimationFrame(() => {
    window.scrollTo(0, parseInt(saved, 10));
  });
}, [navigationType, isLoading]);
```

## 六、经验总结

1. **Masonry 行间隙产生"白色溢出"的根因**：`gridRowEnd: span N` 的 N 包含 gap 行，多出的空间显示父容器背景色。解决方法：让父容器背景透明，或将视觉样式（border-radius、background）下移到图片容器层。

2. **React Router 路由切换期间 scrollY 会被浏览器压缩**：新页面 DOM 上树后，浏览器立即限制 scrollY 至新页面最大可滚动值。组件 unmount cleanup 在此之后才运行，因此不能在 cleanup 里读取"用户离开时"的 scrollY。正确做法：scroll 事件实时持久化。

3. **圆角剥离原则**：使用 `overflow: hidden + border-radius` 做圆角裁切时，若父容器比内容高（如 masonry gap），裁切效果会作用于父容器边缘而非图片边缘，导致图片角看起来是平角。应在"直接包裹图片的容器"上设置 `border-radius`，不依赖上层父容器的 overflow 裁切。
