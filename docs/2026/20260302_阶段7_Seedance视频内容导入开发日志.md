## 三、踩坑与修复

### 问题：原导入脚本使用顺序编号匹配

**根因**：`importSeedance.js` 解析 README 时用 `currentNumber++` 顺序编号（1, 2, 3...），然后尝试与 `video-urls.json` 的 key（YouMind ID 如 "733", "731"）匹配。两套编号完全对不上，导致所有 529 条记录都回退到占位内容。

**表现**：前端显示 "Seedance Prompt #730"、"Video generation prompt #730" 等占位文本。

**修复**：重写 `parseReadme()` 函数，从每个 `###` section 中提取 `youmind.com/...?id=XXX` 链接中的 ID，用该 ID 作为匹配 key。

```javascript
// 修复前：顺序编号
currentNumber++;
matchedPrompt = parsedPrompts.find(p => p.number === parseInt(promptId));

// 修复后：YouMind ID 匹配
const idMatch = section.match(/youmind\.com[^\)]*[?&]id=(\d+)/);
const youmindId = idMatch[1];
promptMap[youmindId] = { title, prompt, ... };
```

### 其他改进

1. **去重提示词文本**：README 中部分提示词代码块存在重复，使用 `Set` 去重
2. **精选标记**：有完整提示词的 100 条标记为 `isFeatured: true`
3. **增强分类**：扩展 `guessCategory()` 关键词列表，覆盖更多类型
4. **增强标签**：扩展 `extractTags()` 关键词，添加 fpv/drone/pov/cyberpunk 等
5. **`--clear` 参数**：支持先清空数据库再导入
6. **提取 sourceUrl**：带完整提示词的记录使用原文 Source 链接，其他使用 YouMind 链接

# 阶段7.5 — Seedance 视频代理 & 自动同步

**日期**：2026-03-03  
**分支**：main  
**核心变更**：解决 Twitter 视频 CORS 播放问题，添加后端代理 + 前端适配 + YouMind 自动同步脚本

---

## 一、问题背景

| 视频源 | 能播放？ | 有声音？ |
|--------|----------|----------|
| GitHub Releases | ✅ 可跨域播放 | ❌ 无音轨 |
| Twitter/X (twimg.com) | ❌ CORS 限制 | ✅ 有声音 |

**目标**：让 Twitter 视频通过后端代理播放，既能播又有声音。

---

## 二、实现方案

### 1. 后端视频代理接口

`GET /api/seedance/proxy-video?url=xxx`

- 白名单验证域名（`video.twimg.com`、`pbs.twimg.com`、`github.com`）
- 服务器端 `fetch` Twitter 视频并 Stream 转发
- 支持 Range 请求（进度条 seek）
- 设置 `Cross-Origin-Resource-Policy: cross-origin` 允许跨域嵌入
- 缓存 1 小时 (`Cache-Control: public, max-age=3600`)

### 2. 前端代理适配

新增 `getVideoSrc(url)` 工具函数：
```javascript
// client/src/services/seedanceApi.js
export const getVideoSrc = (url) => {
    if (!url) return '';
    if (url.includes('twimg.com')) {
        return `${baseURL}/seedance/proxy-video?url=${encodeURIComponent(url)}`;
    }
    return url;
};
```

`VideoCard.js` 和 `SeedanceDetail.js` 的 `<video src>` 均使用 `getVideoSrc()` 包装。

### 3. YouMind 自动同步脚本

`server/scripts/syncSeedanceFromYouMind.js`

功能：
- 从 YouMind 下载最新 CSV 导出（含完整提示词 + Twitter 视频 URL）
- 智能 Upsert：按 `sourceId` 去重，只更新有变化的字段，保留用户交互数据
- 支持 `--dry-run`（预览不写入）和 `--download-only`（仅下载 CSV）
- CSV 备份到 `_data_sources/seedance/csv_backups/`

```bash
# 预览
node scripts/syncSeedanceFromYouMind.js --dry-run

# 正式同步
node scripts/syncSeedanceFromYouMind.js

# 仅下载 CSV
node scripts/syncSeedanceFromYouMind.js --download-only
```

## 三、踩坑与修复

### Bug 1：Express 路由顺序

**问题**：`/proxy-video` 定义在 `/:id` 之后，Express 把 `proxy-video` 当作 `:id` 参数匹配，导致代理接口永远不会被触发。

**修复**：将 `/proxy-video` 路由移到 `/:id` 之前。

```javascript
// ❌ 错误顺序（原来）
router.get('/:id', ...)     // 匹配 "proxy-video" 作为 id
router.get('/proxy-video', ...) // 永远不会到达

// ✅ 正确顺序（修复后）
router.get('/proxy-video', ...) // 精确匹配优先
router.get('/:id', ...)          // 通配在后
```

### Bug 2：跨域资源策略

**问题**：前端 (port 3100) 请求后端 (port 5500) 的代理视频时，浏览器报 `ERR_BLOCKED_BY_RESPONSE.NotSameOrigin`。

**修复**：在代理响应中添加：
```javascript
res.setHeader('Cross-Origin-Resource-Policy', 'cross-origin');
res.setHeader('Access-Control-Allow-Origin', '*');
```
