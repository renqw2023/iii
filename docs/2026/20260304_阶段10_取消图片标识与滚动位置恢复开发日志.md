# 阶段10：取消图片标识 & 瀑布流滚动位置恢复

**日期**: 2026-03-04  
**开发者**: AI Assistant

---

## 一、需求背景

用户反馈两个体验问题：

1. **图片标识冗余**：`/gallery` 和 `/explore` 页面卡片左上角的模型标识（如 "NanoBanana Pro"、"GPT Image"、"--sref xxx"）占用视觉空间，希望取消显示。

2. **瀑布流返回位置丢失**：`/gallery`、`/explore`、`/seedance` 三个页面均采用无限滚动瀑布流布局。当用户下拉到很深的位置后，点击某个卡片进入详情页，再点击返回按钮，页面会回到顶部，用户需要重新下拉寻找之前浏览到的内容，体验非常差。

---

## 二、技术分析

### 问题1：图片标识

- `GalleryCard.js` 中第165-170行渲染了 `gallery-model-badge` 元素
- `SrefCard.js` 中第88-91行渲染了 `liblib-style-tag` 元素

**方案**：直接删除相关 JSX 代码块。

### 问题2：滚动位置丢失

**根本原因分析**：

1. `ScrollToTop.js` 组件在**每次路由变化时**无条件调用 `window.scrollTo(0)`，包括浏览器后退（POP）操作
2. 详情页返回按钮使用 `navigate('/gallery')` 等硬导航（PUSH），而非浏览器后退
3. React 组件在路由切换时会重新挂载，导致浏览器原生的 scroll restoration 不可靠
4. `react-query` 的 `useInfiniteQuery` 虽然有缓存，但组件重建后 DOM 重新渲染，scrollY 无法由浏览器自动恢复

**方案**：采用 `sessionStorage` 主动保存/恢复滚动位置：

```
用户点击卡片 → 保存 window.scrollY 到 sessionStorage
                → navigate 到详情页 (PUSH)
                → ScrollToTop 即时滚到顶部

用户点击返回   → navigate(-1) 触发浏览器后退 (POP)
                → ScrollToTop 检测到 POP，跳过滚顶
                → 列表页挂载，react-query 返回缓存数据
                → useEffect 检测到 POP + sessionStorage 有值
                → 100ms 延迟后恢复 scrollTo(savedPosition)
```

## 四、关键代码说明

### 4.1 ScrollToTop 改造

```javascript
import { useLocation, useNavigationType } from 'react-router-dom';

const ScrollToTop = () => {
  const { pathname } = useLocation();
  const navigationType = useNavigationType();

  useEffect(() => {
    // POP = 浏览器后退/前进 → 保持原有滚动位置
    if (navigationType === 'POP') return;
    // PUSH / REPLACE → 即时滚到顶部
    window.scrollTo({ top: 0, left: 0, behavior: 'instant' });
  }, [pathname, navigationType]);

  return null;
};
```

### 4.2 卡片组件 — 保存滚动位置

```javascript
// GalleryCard.js 示例
onClick={() => {
    sessionStorage.setItem('gallery_scroll', String(window.scrollY));
    navigate(`/gallery/${prompt._id}`);
}}
```

### 4.3 列表页 — 恢复滚动位置

```javascript
const scrollRestoredRef = useRef(false);
const navigationType = useNavigationType();

useEffect(() => {
    if (scrollRestoredRef.current) return;    // 只恢复一次
    if (navigationType !== 'POP') return;     // 仅后退时恢复
    if (isLoading) return;                    // 等待数据加载完成

    const savedScroll = sessionStorage.getItem('gallery_scroll');
    if (!savedScroll) return;

    scrollRestoredRef.current = true;
    sessionStorage.removeItem('gallery_scroll');

    // 等待 DOM + 图片占位渲染后恢复
    setTimeout(() => {
        window.scrollTo(0, parseInt(savedScroll, 10));
    }, 100);
}, [navigationType, isLoading]);
```

## 五、设计考虑

1. **为什么用 `sessionStorage` 而不依赖浏览器原生 scroll restoration?**
   - React SPA 中路由切换导致组件重新挂载，DOM 完全重建
   - 浏览器的原生 scroll restoration 在 DOM 重建场景下不可靠
   - `sessionStorage` 在浏览器标签页生命周期内持久化，关闭标签页自动清除

2. **为什么使用 `navigate(-1)` 而非 `navigate('/gallery')`?**
   - `navigate(-1)` 触发 POP 导航类型，与 `ScrollToTop` 配合跳过滚顶
   - `navigate('/gallery')` 触发 PUSH，会被 `ScrollToTop` 滚顶，且不携带之前的 URL 参数状态

3. **100ms 延迟的作用**
   - react-query 缓存数据虽然同步可用，但 React 渲染 DOM 需要一帧时间
   - 100ms 确保瀑布流卡片的占位高度已计算完成（`gridRowEnd: span`）
   - 浏览器缓存的图片加载几乎是即时的，不需要更长延迟

4. **`scrollRestoredRef` 的作用**
   - 防止 `useEffect` 因依赖变化被多次触发时重复恢复
   - 确保只在首次满足条件时恢复一次
