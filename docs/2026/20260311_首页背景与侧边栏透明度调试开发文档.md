# 20260311 首页背景与侧边栏透明度调试开发文档

## 1. 需求背景

本次调试围绕首页和全局布局的背景层级展开，目标是快速回答并验证以下几个问题：

- 项目当前“全局背景色”到底是什么
- 为什么首页截图看起来是淡紫色，而不是纯白
- 是否可以临时关闭首页淡紫色背景，直接观察纯白版本
- 页脚和左侧边栏是否能改成更白的底色
- 侧边栏降低不透明度后，视觉上为什么几乎没有变化
- 打开动态背景层后，透明侧边栏是否会更容易看出层次

本次工作属于视觉调试和样式验证，不涉及新的业务功能。

## 2. 问题定位过程

### 2.1 全局背景色

先检查了全局主题变量和基础样式：

- `client/src/styles/theme-variables.css`
- `client/src/index.css`

确认结果：

- 全局基础底色变量 `--bg-primary` 在亮色主题下为 `#ffffff`
- `body` 和 `#root` 的 `background-color` 使用的是 `var(--bg-primary)`

因此，从“全局页面容器”的角度看，项目默认背景色确实是白色。

### 2.2 首页为什么看起来是紫色

继续检查首页结构和首页专属样式：

- `client/src/pages/Home.js`
- `client/src/styles/gallery.css`

定位到首页主体内容包裹层：

- `Home.js` 中使用了 `.home-dark-area`
- `.home-dark-area` 的背景来自 `var(--page-bg)`

而 `--page-bg` 在亮色主题下被定义为淡紫色渐变：

```css
linear-gradient(135deg, #e8edff 0%, #f3eeff 60%, #fdf0ff 100%)
```

这就解释了为什么首页截屏看上去是淡紫色：

- 不是 `body` 的白底失效了
- 而是首页主体区域额外覆盖了一层淡紫色渐变背景

## 3. 本次实际改动

### 3.1 暂时关闭首页渐变背景

为了便于直接观察纯白版本，先将：

- `client/src/styles/gallery.css`

中的 `.home-dark-area` 背景从：

```css
background: var(--page-bg);
```

改成：

```css
background: var(--bg-primary);
```

效果：

- 首页主体区域不再显示淡紫色渐变
- 视觉上会回到更接近全局白底的状态

### 3.2 将页脚改为白色背景

为了统一底部视觉，将：

- `client/src/components/Layout/Footer.js`

中的页脚背景从：

```js
backgroundColor: 'var(--bg-secondary)'
```

改为：

```js
backgroundColor: 'var(--bg-primary)'
```

说明：

- 原来的 `var(--bg-secondary)` 在亮色主题下是 `#f8fafc`
- 这是一种很浅的灰白，不是纯白
- 改成 `var(--bg-primary)` 后，页脚更接近纯白

### 3.3 将桌面侧边栏改为半透明白

继续调整：

- `client/src/components/Layout/Sidebar.js`

将侧边栏主容器背景从主题变量改为显式透明白。

调试过程分两步：

1. 先测试 `rgba(255, 255, 255, 0.7)`
2. 再继续测试 `rgba(255, 255, 255, 0.5)`

最终当前保留值为：

```js
background: 'rgba(255, 255, 255, 0.5)'
```

也就是 50% 透明白。

## 4. 为什么侧边栏改成 50% 透明后几乎没效果

这是本次调试里最关键的结论之一。

### 4.1 原因一：侧边栏后面的底本来就很浅

侧边栏后方区域本身并不是深色或高对比背景，而是接近白色的页面底层。

因此即使把侧边栏改成：

```css
rgba(255, 255, 255, 0.5)
```

透出来的也仍然是浅色内容，肉眼不容易看出明显差异。

### 4.2 原因二：动态背景层默认并没有开启

检查了以下文件：

- `client/src/components/Layout/Layout.js`
- `client/src/components/UI/MeshBackground.js`
- `client/src/components/UI/MeshBackground.css`

发现：

- `Layout.js` 中虽然挂载了 `<MeshBackground />`
- 但 `MeshBackground` 组件默认参数是 `enabled = false`

也就是说：

- 动态背景系统代码已经存在
- 但默认处于关闭状态
- 所以侧边栏透明后，并没有明显的彩色底层透出来

这就是“透明度改了，但看起来没什么变化”的根本原因。

## 5. 动态背景的临时验证

为了验证上述判断，临时在：

- `client/src/components/Layout/Layout.js`

中把：

```jsx
<MeshBackground />
```

改成：

```jsx
<MeshBackground enabled />
```

目的只有一个：

- 让全局动态 mesh 背景显示出来
- 配合 50% 透明侧边栏，观察是否更容易看到透出效果

验证结论：

- 打开后，透明侧边栏的层次感明显更容易被感知
- 说明之前“不明显”并不是透明度设置失效
- 而是因为被透出的底层本来就没有足够强的视觉信息

## 6. 当前最终保留状态

根据最后一轮确认，“可以了，先关闭”，因此动态背景没有保留开启状态。

当前实际保留如下：

### 保留

- 首页 `.home-dark-area` 维持纯白背景
- 页脚维持白色背景
- 侧边栏维持 `rgba(255, 255, 255, 0.5)` 半透明白

### 关闭

- 全局 `MeshBackground` 重新恢复为默认关闭状态

也就是说，当前仓库状态是：

- 不显示动态 mesh 背景
- 但保留了侧边栏半透明白和首页/页脚的白底调整

## 8. 后续建议

如果后续继续做视觉优化，推荐按下面顺序继续：

### 方案 A：给侧边栏增加毛玻璃感

可以在侧边栏主容器上增加：

- `backdrop-filter: blur(...)`
- `-webkit-backdrop-filter: blur(...)`

这样即使背景本身颜色不强，视觉上也会比单纯透明更容易看出“浮层感”。

### 方案 B：只在特定页面开启动态背景

不要全站打开，而是：

- 仅首页开启
- 或仅 Explore / Gallery / Seedance 页面开启

这样可以在保留整体稳定性的同时，为重点页面增加层次感。

### 方案 C：把透明度和背景效果做成主题开关

例如后续可以增加：

- 标准白底模式
- 柔和半透明模式
- 动态背景实验模式

这样可以降低反复手工改代码试样式的成本。

## 10. 总结

本次调试最终确认了三件事：

1. 全局底色本来是白色，但首页曾被 `.home-dark-area` 的淡紫色渐变覆盖
2. 侧边栏透明度变化不明显，不是样式失效，而是底层背景默认信息太弱
3. 打开动态背景后，透明侧边栏效果会更明显，但当前按要求已恢复为关闭状态

当前保留方案更偏向稳妥：

- 首页白底
- 页脚白底
- 侧边栏 50% 半透明白
- 动态背景代码保留但默认关闭
