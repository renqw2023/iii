## 一、背景与动机

### 1.1 历史问题

阶段 10 之前，三个详情页（Gallery / Explore / Seedance）均采用**页面跳转**方式打开：

```
/gallery       → GalleryList  (组件挂载)
/gallery/:id   → GalleryDetail (GalleryList 卸载，GalleryDetail 挂载)
← 返回         → GalleryList  (重新挂载，图片 DOM 全部销毁重建)
```

这导致：
- **返回时滚动位置丢失**：组件重新挂载，浏览器默认跳回顶部
- **补救逻辑复杂**：经历 3 次迭代（sessionStorage + data-card-id + requestAnimationFrame 轮询）才勉强稳定
- **闪烁问题**：瀑布流图片 DOM 销毁重建有明显闪烁

### 1.2 根本解法

改为 **嵌套路由 + createPortal Modal** 方案：

```
/gallery       → GalleryList (Outlet 空，Modal 未渲染)
/gallery/:id   → GalleryList (始终挂载) + GalleryModal (Portal 到 body)
← 返回         → GalleryList (从未卸载，滚动位置天然保留)
```

**核心原理**：列表组件从不卸载，打开详情只是在 `body` 上叠加一个弹层，关闭弹层后列表状态完全一致。

## 二、技术方案

### 2.1 路由结构变更

```jsx
// Before — 平级路由
<Route path="gallery" element={<GalleryList />} />
<Route path="gallery/:id" element={<GalleryDetail />} />

// After — 嵌套路由
<Route path="gallery" element={<GalleryList />}>
  <Route path=":id" element={<GalleryModal />} />
</Route>
```

同样处理 `/explore` 和 `/seedance`。

React Router 嵌套路由的渲染机制：
- 访问 `/gallery` → 渲染 `GalleryList`，其内部 `<Outlet />` 为空
- 访问 `/gallery/:id` → 渲染 `GalleryList` + 将 `GalleryModal` 插入 `<Outlet />` 位置
- `GalleryModal` 内部用 `createPortal` 将内容传送到 `document.body`，与 DOM 层级无关

### 2.2 Modal 关闭逻辑

```jsx
const handleClose = () => {
  if (location.state?.fromList) navigate(-1);  // 从列表来 → 后退
  else navigate('/gallery');                    // 直接访问 → 回列表首页
};
```

卡片点击时传入 state：
```jsx
onClick={() => navigate(`/gallery/${prompt._id}`, { state: { fromList: true } })}
```

### 2.3 UI 设计参考

参考 [meigen.ai](https://www.meigen.ai/) 的详情弹层设计：
- **左侧**：大图/视频占满（58% 宽度），黑色背景 + `object-fit: contain`
- **右侧**（42%）：分三层
  - `header`：模型标签 / sref 代码 + 关闭按钮
  - `body`：可滚动，标题 → 统计 → 提示词 → 标签 → 来源
  - `footer`：固定底部操作栏（Copy Prompt 主按钮 + Like / Bookmark / Share 图标按钮）

多媒体处理（SrefModal）：
- **1 张图**：铺满左侧面板
- **多张图/视频**：主图大视图 + 底部缩略图条，点击切换

## 四、各文件详细改动说明

### 4.1 App.js

```jsx
// 删除
import SrefDetail from './pages/SrefDetail';
import GalleryDetail from './pages/Gallery/GalleryDetail';
import SeedanceDetail from './pages/Seedance/SeedanceDetail';

// 新增
import SrefModal from './pages/SrefModal';
import GalleryModal from './pages/Gallery/GalleryModal';
import SeedanceModal from './pages/Seedance/SeedanceModal';

// 路由由平级改为嵌套
<Route path="explore" element={<Explore />}>
  <Route path=":id" element={<SrefModal />} />
</Route>
<Route path="gallery" element={<GalleryList />}>
  <Route path=":id" element={<GalleryModal />} />
</Route>
<Route path="seedance" element={<SeedanceList />}>
  <Route path=":id" element={<SeedanceModal />} />
</Route>
```

### 4.2 ScrollToTop.js

```jsx
// 新增 isModalRoute 判断
const isModalRoute = /\/(gallery|explore|seedance)\/[^/]+/.test(pathname);
if (navigationType === 'POP' || isModalRoute) return;
```

逻辑：Modal 路由打开时（PUSH 导航到 `/gallery/:id`），列表页保持挂载，不应滚顶。

### 4.3 列表页（GalleryList / Explore / SeedanceList）

**删除的代码**（每个列表页约 35 行）：
```jsx
// 删除 scrollRestoredRef
const scrollRestoredRef = useRef(false);

// 删除 sessionStorage 滚动恢复 useEffect（共约 35 行）
useEffect(() => {
  const savedScrollId = sessionStorage.getItem('gallery_scroll_id');
  if (scrollRestoredRef.current) return;
  if (navigationType !== 'POP') return;
  // ... requestAnimationFrame 轮询查找卡片 ...
}, [navigationType, isLoading]);
```

**新增**：
```jsx
// import 加 Outlet
import { ..., Outlet } from 'react-router-dom';

// JSX 末尾加
<Outlet />
```

`navigationType === 'POP'` 检测仍保留在 URL 同步 useEffect 中，原因：
- Modal 关闭时（navigate(-1)）是 POP 导航，此时 searchParams 已由 history 恢复，不需要重新 setSearchParams

### 4.4 卡片组件（GalleryCard / SrefCard / VideoCard）

```jsx
// Before
onClick={() => {
  sessionStorage.setItem('gallery_scroll_id', prompt._id);
  navigate(`/gallery/${prompt._id}`);
}}
// data-card-id={prompt._id}  ← 也删除

// After
onClick={() => navigate(`/gallery/${prompt._id}`, { state: { fromList: true } })}
```

### 4.5 Modal 组件共同结构

```jsx
// 三个 Modal 均包含：
// 1. body 滚动锁定
useEffect(() => {
  document.body.style.overflow = 'hidden';
  return () => { document.body.style.overflow = ''; };
}, []);

// 2. ESC 键关闭
useEffect(() => {
  const handler = (e) => { if (e.key === 'Escape') handleClose(); };
  document.addEventListener('keydown', handler);
  return () => document.removeEventListener('keydown', handler);
}, []);

// 3. createPortal 渲染到 body
return createPortal(<>...</>, document.body);

// 4. Helmet SEO（modal 打开时覆盖页面 title / meta）
<Helmet>
  <title>{prompt.title} - AI Prompts Gallery</title>
  <meta name="description" content={...} />
  <meta property="og:image" content={...} />
</Helmet>
```

## 六、各 Modal 设计差异

### 6.1 GalleryModal（图片类）

- 左侧：单张预览图，`object-fit: contain`，点击可放大（lightbox 预留）
- 右侧 body：模型标签 → 标题 → 统计 → prompt 文本框 → 标签 → 来源
- 右侧 footer：**Copy Prompt**（主）+ Like + Bookmark + Share

### 6.2 SrefModal（风格参考类，可含多图+视频）

- 左侧（1 媒体）：直接铺满，图片可点击放大（lightbox）
- 左侧（多媒体）：主视图 + 底部缩略图条，缩略图支持图片和视频切换
- 右侧 body：sref 代码 → 标题 → 统计（含 N/M 计数器）→ 描述 → 标签
- 右侧 footer：**Copy --sref**（主）+ Like

### 6.3 SeedanceModal（视频类）

- 左侧：视频播放器（autoPlay + loop），占满左侧面板
- 右侧 body：分类标签 → 标题 → 统计（含作者）→ prompt 文本框（含翻译功能）→ 作者/来源链接 → 标签
- 右侧 footer：**Copy Prompt**（橙红渐变主）+ Like + Bookmark + Share

## 七、解决的历史问题对照

| 问题 | 原方案 | 新方案 |
|------|--------|--------|
| 返回后闪烁 | 列表重新挂载，图片重新加载 | 列表从不卸载，0 闪烁 |
| 滚动恢复 | sessionStorage + requestAnimationFrame 轮询 50 次 | 无需任何代码，天然保留 |
| 代码复杂度 | 3 个列表页各 ~35 行恢复逻辑 | 全部删除 |
| 直接访问 URL | 进入详情页面，返回回列表顶部 | 进入列表+Modal，关闭 Modal 留在原位（navigate('/gallery')） |
| SEO | 详情页完整 HTML | Modal 打开时 Helmet 覆盖 title/description/og:image |
