## API 技术方案

### Wan2.7 Image — DashScope 异步任务

```
POST https://dashscope.aliyuncs.com/api/v1/services/aigc/text2image/image-synthesis
Headers: Authorization: Bearer {DASHSCOPE_API_KEY}
         X-DashScope-Async: enable
Body: { model: "wan2.7-image", input: { prompt }, parameters: { size: "1024*1024" } }

→ Response: { output: { task_id: "..." } }

轮询: GET https://dashscope.aliyuncs.com/api/v1/tasks/{task_id}
→ { output: { task_status: "SUCCEEDED", results: [{ url: "..." }] } }
```

超时: 2 分钟，轮询间隔: 3 秒

### Wan2.7 Video — DashScope 异步任务

```
POST https://dashscope.aliyuncs.com/api/v1/services/aigc/video-generation/video-synthesis
Headers: Authorization: Bearer {DASHSCOPE_API_KEY}
         X-DashScope-Async: enable
Body: { model: "wan2.7-t2v", input: { prompt, img_url? }, parameters: { resolution: "1280*720", duration: 5 } }

→ Response: { output: { task_id: "..." } }

轮询: GET https://dashscope.aliyuncs.com/api/v1/tasks/{task_id}
→ { output: { task_status: "SUCCEEDED", video_url: "..." } }
```

超时: 6 分钟，轮询间隔: 4 秒

### Veo 3.1 — Vertex AI 长时操作 (LRO)

```
POST https://{LOCATION}-aiplatform.googleapis.com/v1beta1/
     projects/{PROJECT_ID}/locations/{LOCATION}/
     publishers/google/models/veo-3.1-generate-001:generateVideo
Headers: Authorization: Bearer {GOOGLE_OAUTH2_TOKEN}
Body: { instances: [{ prompt, image?: { uri } }],
        parameters: { aspectRatio: "16:9", durationSeconds: 8,
                      generateAudio: false, resolution: "720p" } }

→ Response: { name: "projects/.../operations/..." }

轮询: GET https://{LOCATION}-aiplatform.googleapis.com/v1beta1/{OPERATION_NAME}
→ { done: true, response: { generateVideoResponse: { generatedSamples: [{ video: { uri: "gs://..." } }] } } }

GCS 下载: GET https://storage.googleapis.com/storage/v1/b/{BUCKET}/o/{ENCODED_OBJ}?alt=media
Auth: Bearer {GOOGLE_OAUTH2_TOKEN}
```

超时: 10 分钟，轮询间隔: 10 秒（Veo 生成慢）

**认证**: `google-auth-library` (已安装 v10.6.1) + `GOOGLE_APPLICATION_CREDENTIALS` service account JSON

## 关键架构决策

### 1. videoService.js 多 provider dispatch

`generateVideo()` 函数新增 provider dispatch，根据 `model.provider` 路由:
```js
if (model.provider === 'dashscope') return generateWanVideo(...)
if (model.provider === 'vertexai')  return generateVeoVideo(...)
// default: Seedance (ByteDance Volcano Ark)
```

### 2. Wan2.7 Video 固定积分 vs Seedance 按秒计费

Wan2.7 每次生成一个固定时长 clip（不像 Seedance 可配 4-12s），
因此用 `creditFlat` 字段而非 `PER_SEC_RATES`，`getCreditCost` 函数优先检查 `model.creditFlat`。

### 3. Veo 按秒积分 + 额外音频费率

Veo 音频价格独立（不是 Seedance 的 +30% 倍率），因此用单独的 `VEO_RATES` 对象:
```js
VEO_RATES = { 'veo-3-1': { noAudio: 30, audio: 58 }, 'veo-3-1-fast': { noAudio: 15, audio: 22 } }
```

### 4. Veo GCS 视频下载

Vertex AI Veo 输出 GCS URI (`gs://bucket/path/video.mp4`)，通过 GCS JSON API + OAuth2 token 下载到本地 `uploads/generated/`，无需额外 npm 包。

### 5. Wan2.7 Video 子模式 UI

前端一个 `wan2-7` 模型按钮 → 选中后显示 Sub Mode 选择器（4个子模式）。
- `t2v` / `i2v` — 可用
- `r2v` / `videoedit` — Phase 2（标记 Soon，需视频上传 UI）

生成时将 `wan2-7` 映射为实际子模型 key (`wan2-7-t2v` 等) 发送给后端。

## 经验总结

1. **DashScope 异步任务模式**与 Seedance 的 Volcano Ark 接近，但端点不同（`/api/v1/tasks/` vs `/api/v3/contents/generations/tasks/`），轮询 key 也不同（`task_status` vs `status`）
2. **Vertex AI LRO** 需要 token 刷新策略（每次 poll 重新获取 token），因为 Veo 生成可能超过 30 分钟（罕见），但 token 1 小时过期
3. **Veo GCS 输出** 不需要 `@google-cloud/storage` SDK，直接用 GCS JSON API + OAuth2 token 下载即可
4. **前端 getVideoCost** 必须在组件内定义（不能提到组件外），因为它依赖 `modelKey` 和 `wanSubMode` 状态变量
